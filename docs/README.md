# 文档说明

建议按下面顺序阅读：

1. [前期调研](./model-research.md)
2. [功能需求草案](./prd.md)
3. [架构说明](./architecture.md)
4. [接口文档](./api-reference.md)
5. [环境变量说明](./env.md)
6. [项目整体路线](./roadmap.md)

当前文档分工如下：

- `model-research.md`：记录模型与平台调研结论。
- `prd.md`：记录项目目标、首版范围和非目标。
- `architecture.md`：记录系统分层和当前实现方式。
- `api-reference.md`：记录当前已经实现的接口契约。
- `env.md`：记录本地开发需要的环境变量。
- `roadmap.md`：记录整体路线和当前进度。

后续规则：

- 新增接口后，优先更新 `api-reference.md`。
- 新增配置后，优先更新 `env.md`。
- 架构发生变化后，同步更新 `architecture.md`。
